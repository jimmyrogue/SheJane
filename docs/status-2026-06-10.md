## 二、已合并 main、随 v0.1.6 发布(发版进行中)

### 多模型目录重构(`feat/model-catalog`,16 commits,Phase 1–4 全部完成)

fast/deep 层级从全栈消失;用户手选模型或 Auto;详见 [design-model-catalog.md](design-model-catalog.md)。

- **Phase 1 后端**:`model_configs` 加 `description`/`priority`(迁移 013);registry 多行目录(`ListChatModels`/`ResolveModel`/`DefaultChatModelID`);`GET /api/v1/models`;router/计费 rekey 成 model_id;Go 各入口 `mode`→`model`(strict decode 硬 break);seed 目录化;store 双实现 conformance。
- **Phase 2 Anthropic 工具调用**:`CompleteWithTools`(tool_use/tool_result 往返、工具名清洗双向映射、stop_reason 归一化);`max_tokens` 2048 硬编码 → 默认 8192 + 每行可配;`base_url` 可配(代理部署);流式捕获真实 input_tokens(计费精度);admin 目录准入校验(未知 kind 拒绝、chat 模型必须工具可用)。
- **Phase 3 客户端**:`ModeSelector` 目录驱动(Auto + `/api/v1/models`);`ChatMode = 'auto' | <model_id>`;持久化 v2 + 失效回退;daemon `CreateRunRequest.mode`→`model` + `make schemas`;三条发送路径(daemon/云 run/web loop)统一发 `model`。
- **Phase 4 收尾**:admin UI 可编辑 label/description/priority;删除全部死代码(Go `Mode` 类型、`Router.Select`/`NormalizeMode`、daemon `FastDeepRouterMiddleware`、`auto_router.py`);admin providers 状态改为遍历真实目录。

### Auto 任务感知分类器(3 commits)

- 云端 `app.ResolveAutoModel`:对目录候选跑一次小分类调用(**不计费**;10s 超时、goal 截断、单候选短路、spend 限流约束),失败回退默认模型。
- `POST /api/v1/models/resolve`;云 run / daemon run 起点 / web loop 各解析一次,发 `model.selected {resolved_model_id, label, reason}` → 三条路径都点亮「Auto → 模型 · 理由」徽章。
- daemon 持久化解析结果(重启 resume 不丢);解析失败非致命(留 auto,云端按轮映射默认)。

### 一并随 v0.1.6 的早期修复

- **spend 端点限流护栏**:`/agent/llm*`、`/tools/execute`、`/images/*` 共享 120/min per-user 令牌桶(`AGENT_SPEND_RATE_LIMIT_PER_MINUTE` 可调)——web 循环的客户端步数上限不可信,服务端独立兜底。
- **daemon 重启恢复(P0)**:孤儿 run 标 failed;HITL resume 保留档位/workspace/设置/scope=run 授权(`local_runs.mode` 列 + `recover_orphans` + `_hydrate_run_state`)。
- **`run.started` 提前发**(CI 揭出的真问题):原来在 Auto 解析 + agent 构建之后才发,慢机器上快速取消会产生只有 `run.canceled` 的事件流;现在驱动一开始就发,事件顺序修正为 `run.started → model.selected → llm.*`。
