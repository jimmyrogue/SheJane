# Agent 运行时阶段总览

> 更新日期：2026-07-12
> 本文是目标架构阶段编号的唯一来源。当前代码如何运行，以 [`run-loop.md`](run-loop.md) 为准。

## 产品边界

SheJane 的核心将是可独立运行的 Agent 运行内核（Harness Runtime），桌面端只是官方客户端。

- 运行时可以脱离 Electron 独立启动。
- 桌面端只通过公开协议使用运行时，不保留私有调用路径。
- 本地运行时失败时明确报错，不自动改走云端任务。
- 用户可以使用自己的模型密钥、本地模型或第三方中转服务。
- SheJane 自有中转服务如果存在，也只是普通模型供应商，不拥有特殊能力。
- 对话、任务、检查点、工具权限和工作产物由运行时保存。
- 飞书等业务平台连接器不属于运行内核；未来如需接入，应通过标准工具或 MCP 提供，不增加平台专用运行链路。
- Web 聊天和 SheJane Cloud 不再是 Harness 的运行前提。
- 未来移动端只做远程客户端，不在手机上实现完整 Harness。

## 常用术语

文档优先使用中文。代码中的稳定名称保留在括号中：

| 中文名称 | 代码名称 | 含义 |
|---|---|---|
| 运行时 | Runtime | 独立执行 Agent 的本地或远程服务 |
| 客户端 | Client | Desktop、未来移动端、命令行或第三方界面 |
| 命令 | Command | 创建、取消、恢复、审批等幂等请求 |
| 任务 | Run | 一次用户可见任务的完整生命周期 |
| 作业 | Job | 等待工作进程领取的一次可执行工作 |
| 执行尝试 | Attempt | 工作进程领取作业后的一次实际执行 |
| 快照 | Snapshot | 某个资源当前状态的完整视图 |
| 检查点 | Checkpoint | LangGraph 可恢复的图状态 |
| 回执 | Receipt | 命令、模型调用或工具调用的持久结果 |

## 每次修改前必须执行的检查

涉及 Desktop、运行时、任务、事件、模型、工具或检查点时，按以下顺序执行：

1. 从 P1 扫描到 P12，找到最早受影响的阶段。
2. 只指定一个主要阶段，并列出直接上游和下游。
3. 确认该阶段的状态所有者。
4. 对照 `run-loop.md`，分清当前实现与目标设计。
5. 对照优化记录，确认已有的保留、替换和删除决定。
6. 修改后再从 P1 扫描到 P12，检查上下游契约是否仍成立。

修改说明顶部使用下面这个简表即可：

```text
主要阶段：P?.?
上游输入：
下游输出：
状态所有者：
替换的当前路径：
```

研究或设计某个阶段时，依次完成：判断是否必要、确认当前实现、比较成熟方案、确定取舍、安排迁移、验证失败与恢复路径。不必要的层优先删除。

## P1-P12 主链

```text
P1  桌面端连接运行时
 ↓
P2  客户端提交幂等命令
 ↓
P3  运行时认证、校验并原子接收命令
 ↓
P4  客户端读取快照并订阅变化
 ↓
P5  工作进程领取作业并准备执行
 ↓
P6  绑定资源并取得 Agent 定义
 ↓
P7  启动或恢复 LangGraph
 ↓
P8  执行一次模型回合
 ↓
P9  判断模型输出
 ↓
P10 执行工具或等待用户
 ↺ 需要继续时返回 P8
 ↓
P11 结算本次执行并释放资源
 ↓
P12 原子提交结果，客户端更新界面
```

## 状态所有权

| 阶段 | 唯一所有者 | 向下游交付 |
|---|---|---|
| P1 | 桌面连接控制器、运行时会话 | 已初始化的会话和能力 |
| P2 | 客户端只拥有尚未确认的投递记录 | 不可变命令 |
| P3 | 命令日志和运行时数据库事务 | 回执、任务快照、待执行作业 |
| P4 | 运行时保存快照和变化；客户端只保存游标与投影 | 有序变化和临时增量 |
| P5 | 持久作业的租约与代次 | 冻结的执行上下文 |
| P6 | 不可变 Agent 定义；本次执行拥有外部资源 | Agent 定义和已绑定资源 |
| P7 | LangGraph 检查点保存图状态 | 模型、工具、中断等标准输出 |
| P8 | 模型调用账本和助手草稿 | 完整的模型回合 |
| P9 | LangGraph 路由和验证回执 | 工具批次、最终候选或失败 |
| P10 | 工具回执和等待候选 | 有序工具结果或等待候选 |
| P11 | 本次执行的资源清理器 | 已静止的执行结果 |
| P12 | 运行时结果提交事务 | 权威快照、版本和事件游标 |

同一份状态如果同时由两个阶段声称拥有，默认视为边界错误。

## P1：桌面端连接运行时

1. Electron Main 启动。
2. 解析运行时目标：桌面托管的本地运行时，或用户提供的外部运行时。
3. 确定谁负责启动、重启和关闭进程。
4. 本地运行时自行绑定端点，并通过启动通道返回地址和实例信息。
5. 外部运行时直接使用已有地址和凭证。
6. 分别检查进程存活、服务就绪和身份认证。
7. 协商协议版本、客户端信息和能力。
8. Renderer 只获得受控会话，不直接持有长期密钥。
9. 连接失败时进入离线或不兼容状态，不回退云端任务。

输出：一个已认证、已协商能力的运行时会话。

## P2：客户端提交幂等命令

1. 用户提交输入。
2. 客户端只做界面层校验。
3. 生成命令编号和客户端消息编号。
4. 引用已有对话与分支；客户端可建议稳定编号，最终由运行时确认归属和唯一性。
5. 组装文本、附件引用、工作区、模型和运行设置。
6. 声明需要的能力和版本前置条件。
7. 在客户端本地事务中保存待发送命令和临时界面状态。
8. 通过 P1 会话反复发送同一份命令，直到取得回执。

客户端不在每回合上传完整聊天历史，也不拥有权威助手消息。旧版本地对话第一次迁入 Runtime 时可以携带一次历史快照，接纳后以 Runtime 消息为准。

## P3：运行时接收命令

### P3.1 身份认证

- 校验本地会话、配对设备或远程凭证。
- 将凭证映射为运行时内部身份。
- 客户端身份、模型密钥和工具审批相互独立。
- 客户端建议的资源编号进入身份作用域，不共享可抢占的全局命名空间。

### P3.2 请求校验

- 使用有区分字段的稳定请求结构解析 JSON。
- 限制正文大小、字段数量、嵌套深度和标识格式。
- 拒绝未知字段，返回稳定错误码和字段位置。

### P3.3 幂等检查

- 用“用户身份 + 命令编号”查询持久命令日志。
- 相同编号和相同内容返回原回执。
- 相同编号但内容不同返回冲突。

### P3.4 接纳检查

- 检查工作区、对话、父任务和分支权限。
- 检查模型配置、凭证引用和所需能力。
- 冻结本次任务的有效设置。
- 不在这里连接模型、MCP 或其他外部服务。

### P3.5 原子提交

同一数据库事务写入：

- 命令日志与回执；
- 必要的新对话；
- 用户消息；
- 状态为 `in_progress` 的助手消息；
- 状态为 `queued` 的任务；
- 状态为 `pending` 的作业；
- 资源变化记录。

这里会为客户端分配稳定的助手消息对象，但不生成回答内容、不启动 Agent，也不写重复的“任务已创建”事件。

### P3.6 返回回执

返回命令编号、任务编号、资源版本、事件游标和当前任务快照。回执只表示命令已经持久化，不表示工作进程已经开始执行。

## P4：客户端读取快照并订阅变化

1. 认证订阅请求和资源权限。
2. 在一致读取边界取得任务快照和事件高水位。
3. 先发送快照，再回放游标之后的持久变化。
4. 注册只用于唤醒的临时通知，再查一次数据库以关闭竞态窗口。
5. 每个客户端维护自己的游标。
6. 任务状态变化持久化；逐字文本、推理和工具进度只走有界临时通道。
7. 断线后按游标恢复；游标过期时重新读取完整快照。
8. 连接结束不代表任务结束。

数据库是事实来源，通知中心只负责提醒客户端“可能有新数据”。

## P5：工作进程领取并准备执行

### P5.1-P5.3 领取作业

- 先取得本机执行名额，再原子领取作业。
- 写入租约所有者、租约代次、到期时间和尝试次数。
- 首次领取时将任务改为 `running`。
- 同一分支同时最多有一个顶层执行尝试。

### P5.4 准备执行上下文

- 装载权威对话消息、工作区授权和冻结设置。
- 校验模型配置、凭证引用、恢复点和分支关系。
- 重新检查可撤销权限，但不重新选择模型或改写设置。
- 生成只读的执行上下文。

### P5.5-P5.6 监督执行

- 在一个受监督的异步任务中启动图执行、租约心跳和取消监视。
- 所有写入都校验租约所有者和代次。
- 取消必须先持久化请求，再停止模型、MCP 和子进程。
- 等待用户时先保存检查点；P11 清理资源，P12 再结束作业。
- 关停时停止领取新作业，安全任务有限等待，无法确认静止时依靠租约过期恢复。

## P6：绑定资源并取得 Agent 定义

1. 确认执行上下文和租约仍有效。
2. 临时解析模型凭证引用，建立本次模型连接。
3. 建立受能力限制的工作区和环境后端。
4. 装载固定版本的工具、MCP 和子 Agent 清单。
5. 在本次执行作用域中连接所需资源。
6. 根据结构指纹复用或编译 Agent 定义。

Agent 定义只包含图结构、状态结构、静态工具说明、中间件顺序和基础提示词。任务编号、工作区、用户、密钥和可变权限不得进入可复用定义。

## P7：启动或恢复 LangGraph

1. 检查 Agent 定义、状态结构和检查点是否兼容。
2. 明确选择新任务、恢复、中断继续、分支或重放中的一种输入。
3. 解析图线程编号和明确的检查点分支头。
4. 注入只读执行上下文。
5. 使用同步持久化启动事件流。
6. 记录执行开始和新的检查点位置。

产品任务编号与 LangGraph 线程编号不是同一概念。分支使用公开检查点接口，不复制 LangGraph 私有数据库行。

## P8：执行一次模型回合

1. 从图状态和执行上下文构建本轮提示内容。
2. 只加载需要的记忆和技能。
3. 按供应商和工作区策略处理出站数据。
4. 计算上下文、输出、时间和费用预算。
5. 依次删除噪声、转存大结果、取回必要引用，最后才做摘要。
6. 确定本轮可见工具，但只能缩小已授予能力。
7. 预留模型调用预算并调用已绑定的唯一模型。
8. 统一处理文本、推理摘要、工具调用、用量和错误。
9. 更新权威助手草稿；逐字增量仅用于实时显示。
10. 完整消息结束后结算用量，交给 P9。

模型失败时不得静默切换供应商。通用脱敏、工具筛选模型和无引用的直接裁剪不进入默认热路径。

## P9：判断模型输出

1. 校验模型消息、工具编号、停止原因和完整性。
2. 将错误、中止、截断和内容过滤变成明确失败或有界修复。
3. 有完整工具调用时生成工具批次，进入 P10。
4. 没有工具调用时形成最终回答候选。
5. 只按明确的验收要求和持久验证回执判断是否完成。
6. 缺少证据且仍有预算时写入运行时修复指令，再回到 P8。
7. 预算耗尽时返回“验证失败或受阻”，不能假装成功。

计划、进度、拒答和验证不能由多个中间件同时争夺路由权。

## P10：执行工具或等待用户

1. 按模型给出的顺序解析完整工具批次。
2. 在执行前完成结构、能力、撤销状态和工具版本检查。
3. 计算稳定操作编号和参数指纹，查询或创建工具回执。
4. 根据具体参数和资源风险决定允许、询问或拒绝。
5. 任一调用需要询问时，整批在执行前暂停并保存检查点。
6. 生成等待候选，交给 P11 清理资源，再由 P12 原子提交等待状态。
7. 用户决定作为新的幂等命令保存；恢复后重新检查能力、授权和参数。
8. 只并行执行明确标记为安全且资源不冲突的调用。
9. 按工具自身语义处理超时、取消、重试和外部结果核对。
10. 完成工具回执，大结果只保存工作产物引用。
11. 按模型原始顺序写回工具消息并保存检查点，然后返回 P8。

审批不能扩大 P3 已授予的能力。产生副作用的工具在结果不明时先核对，不能盲目重试。

## P11：结算执行并释放资源

1. 接收完成、等待、失败、取消、失去租约或关停结果。
2. 停止接受新的模型、工具和子任务。
3. 等待或取消本次执行拥有的子任务。
4. 将租约代次分类为仍有效或已经失效。
5. 确认检查点位置并核对模型、工具和费用账本。
6. 确认最终回答引用、验证状态和工作产物引用。
7. 关闭模型流、MCP、子进程、临时密钥和监视任务。
8. 生成确定性的执行摘要、清理报告和结构化执行结果。

默认反思模型和“每次任务自动保存目标与回答”的长期记忆写入应删除。统计可以从权威记录计算；显式记忆写入属于工具操作。

## P12：原子提交结果并更新客户端

1. 接收 P11 已静止并已标记租约状态的执行结果。
2. 校验作业、租约、尝试次数和任务版本。
3. 明确决定完成、等待、重试、失败、取消或失去租约。
4. 在一个数据库事务中最终化助手消息，并更新任务、作业和执行尝试。
5. 同一事务绑定用量、工具回执、工作产物和验证结果。
6. 重试时创建下一个待执行作业；失去租约的旧执行不得改写任务。
7. 同一事务追加任务事件和全局资源变化记录。
8. 提交成功后只发送可丢失的唤醒通知。
9. 客户端按快照、版本和游标更新可重建界面。

通知、索引、自动记忆提议、质量评估和分析统计都放在任务提交之后，失败时不得回滚用户任务。

## 研究与实现状态

P1-P12 的必要性、成熟方案对比和目标边界已经完成并锁定。当前代码已完成 P2-P12 的本地 Runtime 主链：可靠命令、本机身份、资源授权、持久作业、租约隔离、供应商绑定、图检查点分支、模型账本、单一完成路由、统一工具执行、资源清理，以及结果和对话投影事务都已落地。Runtime 保存对话正文、版本和变化游标；Electron 可在丢失 SSE 或清空缓存后按分页快照重建界面。编辑重跑保留旧分支，检查点分叉建立新产品对话。桌面端不登录云账号也能完成“模型→工具→模型”循环，daemon 失败不会回退云端，无云执行也不会装配未绑定的云工具。尚未完成的是 P1 的远程认证和移动客户端连接能力；这不影响本机 Runtime 主链。当前完整运行细节以 `run-loop.md` 为准。

后续不再新增阶段研究文档。新的结论直接更新本文和 [`harness-stage-improvement-notes.md`](harness-stage-improvement-notes.md)。
